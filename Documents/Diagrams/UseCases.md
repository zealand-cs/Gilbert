# Usecases 

# USECASE - Create user

###### Precondition:

User want to use the system, and should register as a user.

When new users want to use the program they should register as new user with Email, Password and Name.

###### Postcondition:

Users are now registered in the system and are saved in the database.


# USECASE - Delete user

##### Precondition:

Users no longer wish to use the system and want to delete their user

When users have been active and registered they can now opt out of the service and get their account deleted

##### Postcondition:

Users can now delete their account when they wish to

# USECASE - Edit user

##### Precondition:

A user wish to change their info.

A user might change their name, email, password or so and should be able to edit their info

##### Postcondition:

The user can now edit their own info of their account.

# USECASE - Log in

###### Precondition:

Customers and employees want to log in to the system.

When the customers or employees wish to use the system they should login with their registered login (Email + Password)


###### Postcondition:

The customers and employees can now log in to the system and use the parts of the website they want.


# USECASE - Log out

###### Precondition:

Customers and employees want to log out of the system.

When the customers or employees wish to log out of the system, they should click a logout button or be timed out.


###### Postcondition:

The customers and employees can now log out of the system, when they are done using it.


# USECASE - Create post

##### Precondition:

Customers want to sell their clothes, so they want to be able to put up their clothes for sale by registering it into the system

Customers should be able to register new clothes by taking pictures, answering the questions about condition, receipt, boxes etc.

##### Postcondition:

Customers can now upload images, fill out info and answer questions to start selling their clothes by making a post for each item.

# USECASE - Delete post

##### Precondition:

Customers have an active post but it might be sold or they changed their mind and no longer want to sell.

Customers should have an option to choose to take a post down if something happened to the clothes (sold, destroyed or changed their mind)

##### Postcondition:

Customers can now freely take down a post of theirs by the click of a button

# USECASE - Edit post

##### Precondition:

Customers have an active post but they might want to change the information on the post

Customers can choose to edit their post by changing price, information or conditions of the item

##### Postcondition:

Customers can now freely change their post to reflect the actual value or information of the post.

# USECASE - Role assignment

##### Precondition:

Users who needs different access to different parts of the system.

Users need to be seperated into different roles to seperate from regular members, different sellers, employees and admins.

##### Postcondition:

Users are now assigned different roles with different permission access to the system


# USECASE - Add favorites

##### Precondition:

A user wish to add a post to their favorites

A user can now when viewing posts choose to add to their favorites, so they can view them later or get notifcations (later on)

##### Postcondition:

The user can now add favorites to their profile and see them at any time they want.


# USECASE - Remove favorites

##### Precondition:

A user wish to remove a post from their favorites

The user can now remove a post from their favorites by clicking the button, and no longer be informed on the item.

##### Postcondition:

The user can now freely remove posts as they wish.

# USECASE - See favorites list/view

##### Precondition:

The user has added posts to their favorites and now want to be able to access them.

The user can now click on favorites in the menu and access all their favorited posts.

##### Postcondition:

The user can now access and see a list/view over their favorited posts.

# USECASE - Notifications

##### Precondition:

The user wish to be informed/updated on whats going on.

The user can now get notifications of their favorites being discounted, bids on their posts or chats.

##### Postcondition:

The user now gets notifications of different news

# USECASE - Searching/sorting

###### Precondition:

The user wish to sort/search for specific items.

The user should be able to sort and search for different items, whether its type, brand or size.

###### Postcondition:

The user can now sort and search for the different needs they have.
